Introduction

As music lovers, we are interested in the trending predictions of new music. Therefore, we decided to create a prediction model based on 13 Spotify variables of song features and test it on its music library to decide if it could be a "Hit or a Flop".

To develop our Machine Learning model we used Spotify "hit predictor" dataset, created by Farooq Ansari as our input data.

Metadata Summary

The original data includes 40,000+ songs with release dates ranging from 1960-2019. The dataset includes measurements for the following features for each track:

Danceability, energy, key, loudness, mode, speechiness, acousticness, instrumentalness, liveness, valence, tempo, duration (in milliseconds), time signature, chorus hit and sections.

The last two features were defined by Ansari and extracted from the data received by the API call for Audio Analysis of each particular track, while the first ones are defined by Spotify on its Audio Features API call.

Finally, we use target, a boolean variable describing if the track ever appeared on Billboard's Weekly Hot-100 list (as Ansari obtain in its original dataset), to define if the track could be classified as a hit or flop.

Details about each of the features and what they measure can be found on Kaggle, and through Spotify's documentation. Ansari's original source code can be found here.


Metodology and Results

The team trained and tested 4 different models to evaluate the dataset, in addition to variations within some of the models, like removing some of the variables to prove its significance within each model.

  1. SVM (Support Vector Machine) Model
  2. Logistic Regression
  3. Neural Network/Deep Learning Model
  4. Random Forest Model
Because of the different natures of values that the variables had, we tried 3 kinds of normalization:
  1. 1. Standard Scaler
  2. 2. MinMax Scaler
  3. 3. Independent normalization of each variable
It is important to highlight that the models were run in local servers and then, the ones with the best results were run through Google's Collaboratory to maximize their score.


Results:

After running the 4 models and the variations described above, we chose the Random Forest Model with adjusted settings as the final model, given that it produced the best results with the highest levels of accuracy.
